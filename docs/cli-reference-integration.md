# 命令行参考集成说明
最后更新时间：2026-07-04

`reference/cli-analysis` 目录包含一套命令行代理实现的公开分析包及配套报告。

## 决议
仅将该参考资料用作架构研究依据。所有新增至 VibeLink 的产品代码，均需采用独立洁净开发实现，仅依托通用设计思路、现有公开文档及本项目业务需求开发。

### 可借鉴参考（无版权风险）
- 命令行入口分层架构：快速命令路由前置，再执行完整运行时初始化流程
- 工具运行时规范：包含模式校验、权限校验、进度/结果事件、审计元数据模块
- 权限默认闭守策略：写入、破坏性操作需用户显式确认授权
- 单条命令审批机制：高危 Shell 命令执行前暂停并展示风险说明
- 事件优先执行模型：所有任务/工具操作均生成带游标标识的持久化事件
- 上下文管理方案：会话记录持久化、数据压缩分界、显式内存注入机制

### 禁止复制、内嵌的内容
- `reference/cli-analysis/src` 下所有 TypeScript 源码文件
- 完整的命令/工具实现逻辑、交互提示、界面组件、隐藏功能代码
- 内置工具插件、遥测采集逻辑、私有服务客户端、内部接口交互逻辑

## 已落地洁净开发改造项
首个低风险自研迁移功能：工作区单命令审批能力
1. `src/commandSafety.js`：定义本地风险判定规则，覆盖文件破坏性操作、Git 历史重写、下载即执行、权限提升、内联解释器、网络包安装等高风险场景
2. `src/server.js`：执行 `/api/workspaces/:id/command` 接口前校验风险；若未传入 `approved: true`，高危命令将返回 HTTP 428 状态码并附带结构化风险说明
3. `apps/web/src/main.jsx`：在工作区终端、测试面板内弹出单条命令确认弹窗，用户确认后才重新提交执行
4. 审计事件同步记录两类行为：需审批而被拦截的操作、用户确认后的执行操作，包含命令类型与匹配到的风险规则

## 后续洁净开发阶段规划
1. 制定 VibeLink 专属工具标准规范，覆盖文件编辑、Shell 命令、Git 操作、浏览器操作、审批申请全场景
2. 使用 SQLite 持久化命令审批请求，支持断连后保留审批状态、跨设备处理审批
3. 为各类工具新增独立事件卡片，数据来源为持久化事件条目，而非从会话文本反向推导
4. 新增命令分级标签：只读、工作区写入、联网操作、破坏性操作、高权限操作
5. 基于伪终端（PTY）实现终端会话，高危命令启动前复用同一审批拦截逻辑
6. 基于公开协议与本地清单文件开发插件/MCP 管理能力，不复用参考源码实现

更完整的能力对齐路线见 `docs/cli-capability-alignment-plan.md`。
